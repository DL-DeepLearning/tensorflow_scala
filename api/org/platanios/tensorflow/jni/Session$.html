    Executes a computation in a session.

    Executes a computation in a session.

    The author apologizes for the ugliness of the long argument list of this method. However, take solace in the fact that this is a private method meant to cross the JNI boundary.

    handle

    to the C API TF_Session object (Session.nativeHandle)

    runOptions

    serialized representation of a RunOptions protocol buffer, or null

    inputTensorHandles

    together with inputOpHandles and inputOpIndices specifies the values that are being "fed" (do not need to be computed) during graph execution. inputTensorHandles[i] (which correponds to a Tensor.nativeHandle) is considered to be the inputOpIndices[i]-th output of the Operation inputOpHandles[i]. Thus, it is required that inputOpHandles.length == inputOpIndices.length == inputTensorHandles.length.

    inputOpHandles

    (see inputOpIndices)

    inputOpIndices

    (see inputTensorHandles)

    outputOpHandles

    (see outputOpIndices)

    outputOpIndices

    together with outputOpHandles identifies the set of values that should be computed. The outputOpIndices[i]-th output of the Operation outputOpHandles[i], It is required that outputOpHandles.length == outputOpIndices.length.

    targetOpHandles

    is the set of Operations in the graph that are to be executed but whose output will not be returned

    wantRunMetadata

    indicates whether metadata about this execution should be returned.

    outputTensorHandles

    will be filled in with handles to the outputs requested. It is required that outputTensorHandles.length == outputOpHandles.length.

    returns

    if wantRunMetadata is true, serialized representation of the RunMetadata protocol buffer, false otherwise.
